## 1. 设计哲学

Bloath CMS 面向个人博客内容创作者，设计核心原则是**"界面隐形，内容优先"**。界面元素应尽可能低调，让用户专注于写作和编辑，而非与界面交互。

### 1.1 设计原则

- **扁平优先**：不使用阴影，通过背景色阶和极细分隔线区分层级
- **暖色调**：所有中性色带微弱暖调，避免冷灰带来的生硬感
- **克制用色**：单一品牌色仅用于交互强调，其余保持中性
- **内容密度适中**：留白充分，信息排列有序，不追求高密度
- **结构可见**：网格和边框清晰但不突兀，体现工具的专业性

## 2. 颜色体系

### 2.1 基础色板

```css
:root {
  /* 背景色 */
  --color-canvas: #F7F7F7;      /* 页面画布 - 暖米白 */
  --color-surface: #FFFFFF;     /* 卡片表面 - 纯白 */
  --color-surface-alt: #FCFCFC; /* 次级表面 - 近白 */
  --color-surface-warm: #F0F0EA; /* 暖色面板 - 米灰 */

  /* 文字色 */
  --color-ink: #2E2E2E;         /* 主文字 - 暖近黑 */
  --color-ink-secondary: #6A6972; /* 次要文字 - 中灰 */
  --color-ink-tertiary: #777169;  /* 三级文字 - 暖灰 */
  --color-ink-placeholder: #9CA3AF; /* 占位符 - 浅灰 */

  /* 边框 */
  --color-border: rgba(0, 0, 0, 0.08);    /* 标准边框 */
  --color-border-strong: rgba(0, 0, 0, 0.14); /* 强调边框 */
  --color-border-subtle: rgba(0, 0, 0, 0.04); /* 极细分隔线 */

  /* 品牌色 */
  --color-primary: #207DFF;         /* 主品牌色 - 交互强调 */
  --color-primary-hover: #1A6BDD;   /* 主品牌色悬停 */
  --color-primary-active: #1558B5;  /* 主品牌色按下 */

  /* 语义色 */
  --color-success: #75C33A;         /* 成功 - 绿色 */
  --color-warning: #E0A23A;         /* 警告 - 琥珀色 */
  --color-error: #E5484D;           /* 错误 - 红色 */
  --color-info: #6E8BE0;            /* 信息 - 蓝色 */
}
```

### 2.2 颜色使用规范

| 用途 | 颜色 | 使用场景 |
|------|------|----------|
| 页面背景 | `#F7F7F7` | 整个应用的最外层背景 |
| 卡片/面板 | `#FFFFFF` | 内容容器、侧边栏、编辑器 |
| 主文字 | `#2E2E2E` | 标题、正文、导航文字 |
| 次要文字 | `#6A6972` | 描述文字、元数据、时间戳 |
| 三级文字 | `#777169` | 禁用态、标签、小字说明 |
| 占位符 | `#9CA3AF` | 输入框占位文字 |
| 品牌强调 | `#207DFF` | 按钮、链接、选中态、焦点环 |
| 成功状态 | `#75C33A` | 发布状态、操作成功提示 |
| 错误状态 | `#E5484D` | 删除操作、验证错误 |

### 2.3 颜色禁忌

- 不使用纯黑 `#000000` 作为文字色（使用 `#2E2E2E`）
- 不使用冷灰（如 `#6B7280`、`#9CA3AF` 以外的冷调灰）
- 品牌色不用于边框、分隔线或装饰元素
- 语义色仅用于状态指示，不用于常规 UI 元素

## 3. 字体排印

### 3.1 字体栈

```css
font-family: -apple-system, BlinkMacSystemFont, 'Inter', 'Segoe UI',
             'PingFang SC', 'Microsoft YaHei', sans-serif;
```

- 英文优先使用系统字体栈（Inter 作为回退）
- 中文优先使用苹方（macOS）或微软雅黑（Windows）
- 不加载额外 Web Font，保持轻量

### 3.2 字号与字重

| 角色 | 字号 | 字重 | 行高 | 字间距 | 使用场景 |
|------|------|------|------|--------|----------|
| 显示标题 | 48px | 500 | 1.30 | -1.5px | 营销页面 Hero（不适用 CMS） |
| 页面标题 | 24px | 600 | 1.35 | -0.5px | 大区块标题（不适用 CMS） |
| 卡片标题 | 20px | 600 | 1.40 | normal | 板块标题（不适用 CMS） |
| 导航标签 | 16px | 400 | 1.50 | normal | 导航文字、正文 |
| 正文 | 16px | 400 | 1.50 | normal | 文章内容、描述文字 |
| 按钮文字 | 16px | 500 | 1.00 | normal | 按钮标签 |
| 小字 | 14px | 400 | 1.43 | normal | 元数据、辅助说明 |
| 标注 | 13px | 500 | 1.30 | normal | 标签、状态文字 |
| 超小字 | 12px | 600 | 1.00 | 0.06em | 大写标签、列头（不适用 CMS） |

### 3.3 排版原则

- 标题使用紧凑行高（1.30-1.35），正文使用宽松行高（1.50）
- 标题使用负字间距增强紧凑感，正文保持默认
- 字重分级：400（正文）、500（强调/导航）、600（标题）
- 不使用 700+ 字重（除非营销页面）

- 板块间距：`24px`（中等）/ `48px`（大型）
- 导航项间距：`4px`（紧凑排列）

## 6. 深度与层级

### 6.1 层级系统（扁平化）

| 层级 | 处理方式 | 使用场景 |
|------|----------|----------|
| Level 0（扁平） | 无阴影，无边框 | 页面背景、正文文字 |
| Level 1（色阶） | 背景色切换 | 卡片与画布的区分 |
| Level 2（边框） | `1px solid rgba(0,0,0,0.08)` | 卡片轮廓、分隔线 |

### 6.2 层级原则

- **不使用阴影**：所有层级通过背景色和边框区分
- **色阶过渡**：画布 `#F7F7F7` -> 表面 `#FFFFFF` -> 高亮 `#FCFCFC`
- **边框分隔**：使用极细半透明边框（`rgba(0,0,0,0.08)`）作为主要分隔手段

## 9. 响应式设计

### 9.1 断点定义

| 断点名称 | 宽度 | 设备类型 |
|----------|------|----------|
| `sm` | `< 640px` | 小屏手机 |
| `md` | `640px - 1024px` | 大屏手机 / 平板 |
| `lg` | `> 1024px` | 桌面端 |

### 9.2 各断点布局变化

#### 桌面端（`lg`，`> 1024px`）

- 侧边栏固定显示，宽度 `260px`
- 主内容区内边距 `24px`
- 列表行采用表格布局（多列显示）
- 所有交互元素正常显示

#### 平板端（`md`，`640px - 1024px`）

- 侧边栏可折叠（通过汉堡菜单触发）
- 主内容区内边距 `20px`
- 列表行保持表格布局，但减少列宽分配
- 下拉菜单最大宽度限制为视口宽度 `- 32px`

#### 手机端（`sm`，`< 640px`）

- 侧边栏默认隐藏，通过汉堡菜单触发抽屉式侧边栏
- 侧边栏宽度 `280px`（占屏幕比例更小）
- 主内容区内边距 `16px`
- 列表行切换为卡片布局（单列堆叠）
- 搜索框全宽显示
- 按钮文字缩小至 `13px`
- 表格列隐藏，改用垂直堆叠展示

### 9.3 移动端适配细节

#### 侧边栏抽屉

| 属性 | 值 |
|------|-----|
| 宽度 | `280px` |
| 动画 | `translateX` 滑入/滑出，`200ms` |
| 遮罩层 | `rgba(0, 0, 0, 0.4)` |
| 关闭方式 | 点击遮罩、滑动返回、导航切换 |

#### 列表行移动端布局


- 标题：`14px`，字重 `600`，颜色 `#2E2E2E`
- 副标题：`12px`，颜色 `#9CA3AF`
- 操作按钮：`32x32px` 图标按钮，圆角 `4px`

## 12. 状态反馈规范

### 12.1 加载状态

| 状态 | 处理方式 |
|------|----------|
| 页面加载 | 骨架屏（`#F0F0EA` 背景块，匹配最终布局） |
| 列表加载 | 骨架屏逐行显示 |
| 单个操作 | 按钮内旋转图标（`#207DFF`） |
| 编辑器保存 | Toast 通知（底部右上方，`3s` 自动消失） |

### 12.2 成功/错误反馈

| 状态 | 处理方式 |
|------|----------|
| 保存成功 | Toast：深色背景，绿色前导点，`3s` 消失 |
| 操作成功 | Toast：深色背景，绿色前导点，`5s` 消失 |
| 操作失败 | Toast：深色背景，红色前导点，`5s` 消失 |
| 删除确认 | 对话框确认，危险按钮红色 |

## 13. 无障碍规范

### 13.1 对比度要求

| 组合 | 对比度 | 等级 |
|------|--------|------|
| `#2E2E2E` on `#FFFFFF` | ~14:1 | AAA |
| `#6A6972` on `#FFFFFF` | ~5.5:1 | AA |
| `#207DFF` on `#FFFFFF` | ~4.6:1 | AA |
| `#75C33A` on `#FFFFFF` | ~3.2:1 | AAA（大文本） |

### 13.2 焦点指示

- 所有可交互元素必须支持键盘焦点
- 焦点环：`2px solid rgba(32, 125, 255, 0.20)` + `1px solid #207DFF`
- 焦点环颜色与品牌色一致

### 13.3 触控目标

- 桌面端：最小 `32x32px`
- 移动端：最小 `44x44px`（符合 WCAG 2.1 要求）

### 13.4 减少动画

- 支持 `prefers-reduced-motion: reduce`
- 所有过渡动画在减少动画模式下 Collapse 为即时切换
